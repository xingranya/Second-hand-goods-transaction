# 校园二手交易平台后端说明

## 1. 项目简介
本后端是“校园二手交易平台”的业务服务端，负责认证鉴权、商品管理、订单流转、消息沟通、求购发布、评价管理和后台统计能力。项目保持现有技术栈稳定，不额外引入复杂中间件。

## 2. 技术栈
- `Spring Boot 2.7`
- `Spring Security + JWT`
- `Spring Data JPA`
- `MySQL 8`
- `H2`
  - 用于测试环境
- `Lombok`

## 3. 系统角色
- 普通用户
  - 登录、实名认证、发布商品、发起求购、发送消息、创建订单、完成评价。
- 管理员
  - 登录后台、查看系统统计、管理商品、查看用户、启用/禁用用户、审核认证状态、查看订单。

## 4. 后端模块划分

### 4.1 控制层
- `AuthController`
  - 登录与注册
- `UserController`
  - 当前用户资料与认证
- `ProductController`
  - 商品查询、发布、修改、状态更新
- `OrderController`
  - 前端订单视图与状态推进
- `WantedController`
  - 求购列表与发布
- `MessageController`
  - 会话列表、消息明细、未读统计、已读处理
- `ReviewController`
  - 评价创建、查询、修改、删除
- `SystemController`
  - 数据库健康检查与首页摘要
- `AdminController`
  - 管理端统计、商品管理、用户管理、订单管理

### 4.2 业务层
- `service`
  - 负责封装商品、用户、交易、消息、评价等业务逻辑。

### 4.3 数据访问层
- `repository`
  - 通过 JPA Repository 对用户、商品、订单、消息、求购、评价数据进行持久化访问。

### 4.4 安全层
- `JwtTokenUtil`
  - 负责 JWT 生成与解析
- `JwtAuthenticationFilter`
  - 负责请求鉴权
- `SecurityConfig`
  - 负责路由权限规则与过滤链配置
- `JwtAuthenticationEntryPoint`
  - 负责未登录场景下统一返回认证错误信息

### 4.5 异常处理
- `GlobalExceptionHandler`
  - 统一处理参数校验失败、资源不存在、越权访问和业务异常，输出统一 `{ "message": "..." }` 结构。

## 5. 数据库核心表
- `users`
  - 用户基础信息、认证状态、角色、启用状态
- `products`
  - 商品信息、原价、校区、状态、卖家
- `transactions`
  - 订单参与方、金额、订单状态、时间戳
- `messages`
  - 会话消息、未读状态、关联商品
- `wanted_posts`
  - 求购帖子、期望价格、校区、截止时间
- `reviews`
  - 商品评价、评分、评价人、创建时间

## 6. 核心设计说明

### 6.1 认证与权限
- 登录接口 `POST /api/auth/login` 返回 `token + username + role`
- 用户端通过 `Authorization: Bearer <token>` 访问受保护接口
- `/api/admin/**` 统一要求管理员角色访问
- 普通用户和管理员通过 `role` 字段区分，不扩展复杂 RBAC

### 6.2 DTO 化输出
- 商品、消息、评价、用户资料、后台统计和后台列表接口已统一采用 DTO 返回
- 控制器不再直接暴露 JPA 实体，减少字段泄漏和前端兼容成本

### 6.3 统一异常结构
- 认证失败：`401`
- 权限不足：`403`
- 参数校验失败：`400`
- 资源不存在：`404`
- 统一错误体：

```json
{
  "message": "错误说明"
}
```

## 7. 初始化数据与默认账号

### 7.1 CommandLineRunner 初始化数据
`DemoDataInitializer` 会注入多组初始化数据，包括：
- 用户
- 商品
- 求购帖
- 消息
- 订单
- 评价

### 7.2 默认账号
- 管理员：`admin / 123456`
- 普通用户：`alice / 123456`
- 普通用户：`bob / 123456`

### 7.3 SQL 初始化脚本
- 文件位置：`server/sql/init.sql`
- 适合快速导入 MySQL 初始化环境

## 8. 本地开发与测试

### 8.1 环境要求
- `JDK 17`
- `Maven 3.8+`
- `MySQL 8`

### 8.2 数据库配置
请在 `application.yml` 中配置本地 MySQL 连接信息，并确保数据库名称为 `secondhand`。

### 8.3 常用命令
- 安装与构建：`mvn clean package`
- 运行测试：`mvn test`

## 9. 已覆盖的后端能力
- 登录成功与失败消息统一
- 禁用账号登录拦截
- 当前用户资料聚合统计
- 首页公共摘要接口
- 管理端统计接口
- 管理端商品、用户、订单接口
- 商品 DTO 输出补齐 `campus` 与 `originalPrice`
- 消息、评价接口 DTO 化
- 统一异常输出

## 10. 测试建议
- 认证测试
  - 验证登录成功、错误密码、禁用账号三类场景
- 管理端测试
  - 验证管理员访问成功、普通用户访问被拒绝、商品和用户后台操作落库
- 业务链路测试
  - 验证商品查询、下单、消息沟通、评价创建的核心闭环
