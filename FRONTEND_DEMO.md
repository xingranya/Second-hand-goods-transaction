# 校园二手交易平台前端页面说明

## 1. 文档定位
本说明用于梳理前端页面结构、主要使用流程、默认账号和页面特性。当前前端基于 `Vue 3 + Pinia + Vite + Axios` 实现，视觉风格统一为“校园感清爽高级”。

## 2. 前端模块概览

### 2.1 用户端
- 首页 `/`
  - 展示平台数据摘要、交易流程、热门分类、推荐商品和平台优势。
- 登录页 `/login`
  - 双栏展示布局，包含平台介绍、登录反馈、错误提示和跳转逻辑。
- 搜索页 `/search`
  - 支持关键字、校区、排序、状态筛选，提供加载态、空状态和筛选标签回显。
- 商品详情 `/product/:id`
  - 展示商品信息、卖家信息、评价、聊天与下单入口。
- 发布页 `/publish`
  - 支持发布商品并补齐商品必要字段。
- 消息页 `/messages`
  - 展示会话列表、未读状态和商品上下文消息。
- 求购页 `/wanted`
  - 展示求购广场与求购发布入口。
- 订单页 `/order/:id`
  - 展示订单参与方、商品摘要、订单状态和流转按钮。
- 支付页 `/order/:id/pay`
  - 提供支付确认与订单状态承接流程。
- 评价页 `/order/:id/review`
  - 支持订单完成后的评价提交。
- 最近订单 `/orders/recent`
  - 方便快速进入最近订单链路。
- 个人中心 `/profile`
  - 以工作台方式展示资料、认证状态、统计数据、后台入口和功能导航。
- 认证页 `/verify`
  - 支持学生身份认证信息提交。

### 2.2 管理端
- 管理员登录 `/admin/login`
- 后台概览 `/admin/dashboard`
- 商品管理 `/admin/products`
- 用户管理 `/admin/users`
- 订单管理 `/admin/orders`

管理端采用“最小可用后台”设计，用于处理平台统计、商品、用户与订单管理。

## 3. 默认账号
- 普通用户：`alice / 123456`
- 普通用户：`bob / 123456`
- 管理员：`admin / 123456`

如果使用 `server/sql/init.sql` 初始化数据库，也可使用：
- `seller01 / 123456`
- `buyer01 / 123456`
- `seller02 / 123456`
- `admin / 123456`

## 4. 推荐使用流程

### 4.1 用户端主链路
1. 进入首页，说明平台定位、交易流程和平台数据摘要。
2. 进入搜索页，查看筛选、排序、空状态和商品列表信息。
3. 打开商品详情页，展示商品信息、卖家信息和聊天入口。
4. 从商品详情进入消息沟通或直接下单。
5. 打开订单页，展示订单状态推进。
6. 在订单完成后进入评价页，完成交易反馈闭环。

### 4.2 用户工作台链路
1. 使用普通用户登录。
2. 进入个人中心，展示认证状态、发布数、订单数、未读消息等聚合数据。
3. 切换到“我的订单”“消息中心”“求购广场”等页面，说明系统闭环能力。

### 4.3 管理端链路
1. 使用管理员账号进入 `/admin/login`。
2. 在后台概览页展示用户数、商品数、求购数、订单数、成交数等统计。
3. 进入商品管理，查看上架/下架、删除异常商品等操作。
4. 进入用户管理，查看启用/禁用、认证状态调整等操作。
5. 进入订单管理，展示订单状态检索与异常订单定位。

## 5. 前端亮点
- 登录页不再是单表单，而是承担项目入口、品牌展示和登录反馈三重角色。
- 首页和个人中心从“功能入口集合”升级为“平台总览页”和“用户工作台”。
- 所有核心页面补齐 `loading / empty / error / success` 状态，页面反馈更完整。
- 管理端与用户端保持统一视觉语言，同时通过独立路由和状态管理实现角色隔离。
- 登录后会立即刷新用户资料并联动头部导航，避免页面状态与本地 token 不一致。

## 6. 环境变量
- `VITE_API_BASE_URL`
  - 前端请求基地址，默认值为 `/api`
- `VITE_API_TARGET`
  - 本地开发代理目标地址，默认值为 `http://localhost:8080`

## 7. 构建与校验
- 安装依赖：`npm install`
- 本地开发：`npm run dev`
- 生产构建：`npm run build`

## 8. 使用建议
- 可先从首页和登录页了解平台定位，再进入具体业务流程页面。
- 商品链路建议提前准备 1 到 2 条订单与消息数据，保证订单、评价、消息页面都有可查看内容。
- 管理端建议优先查看统计、商品管理和用户状态管理页面。
